v0.2.28 — Hotfix: Light/System theme now actually applies

Fix

Tapping "Light" or "System" in Profile → Display had no visible effect on
any screen. The setting was being saved correctly, but a force-dark
override at the top of ThemeProvider was firing whenever appState was
ROUTE_PREVIEW or NAVIGATING — and appState persists across screen
transitions. So if you'd just previewed a route and then went to Profile
or History via the bottom nav, every screen rendered dark regardless of
your theme pick.

Now the force-dark only fires when you're actually *viewing* the
map-stage screens (route-preview, navigation). Profile, History,
Community, onboarding, settings — all respect your theme preference.

The bug has been there since the P1-30 design pass; just surfaced now
because tester #9 specifically poked the theme picker.

Test plan

1. Open Profile → Display → tap "Light". Profile background should flip
   to near-white, text to dark immediately.
2. Switch to History via bottom nav — should also be in light theme.
3. Same for Community.
4. Tap "System" — should follow OS theme.
5. Tap "Dark" — should flip back to dark everywhere.
6. Plan a route → preview it → on the route preview screen the theme
   should be dark (intentional, glare reduction). Back-arrow back to
   Route Planning, then Profile → still light if that's your pick.
7. Restart the app — your last pick should persist.
