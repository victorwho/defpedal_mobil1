## TL;DR

**Item count by status (14 total):**
- Already done (full): **0**
- Partially done (foundation in place, finishing work needed): **6** — items 1, 4, 5, 6, 7, 8
- Not started: **8** — items 2, 3, 9, 10, 11, 12, 13, 14

**Top 3 things blocking Phase 1 (Play submission):**
1. **Item 3 — Privacy policy + FAQ correction.** No web URLs published, no mobile screen, and `app/faq.tsx:142–144` actively misrepresents what the app does with GPS. This is the longest critical-path item: legal copy + Romanian translation + 3 web pages + 3 mobile touchpoints.
2. **Item 1 — Account deletion.** Cascade FKs are already shipped (migration `202604200001_cascade_user_fks.sql`), so the SQL surface is small, but the API route, mobile UI, and web fallback page still need to ship together.
3. **Item 4 — Foreground service type.** Tiny code change (one plugin + one permission), but the AndroidManifest is autogenerated on `C:\dpb` so the fix has to live in a config-plugin that the build script picks up. Without this, navigation crashes on Android 14+.

**Recommended week-by-week sequencing for a single developer:**
- **Week 1:** Items 1 (account deletion), 4 (FG service), 5 (release signing), 8 (asset links). All are repo-mostly with verifiable on-device tests; ships an internal alpha.
- **Week 2:** Item 3 (privacy + FAQ + auth footer) — split EN copy first, RO translation second. Item 10 (AAB default + CI guard).
- **Week 3:** Item 7 (UGC moderation: schema → API → mobile UI → ops mailbox). Item 13 retention migrations as an opener (since item 9 depends on it).
- **Week 4:** Item 8 (consent screens + Sentry/PostHog gating). Item 9 (Data Safety form). Item 6 short-term (network_security_config). Closed-test track opens.
- **Week 5+:** Item 11 (dev artefact audit). Item 12 (listing polish, content rating). Item 14 (RO-specific paperwork, DPIA). Item 6 long-term (TLS for OSRM).

**Repo-only vs ops vs out-of-repo split:** ~8 items are repo-only or repo+ops; 4 require web pages (`apps/web/`); 2 require pure out-of-repo (Play Console, ANSPDCP DPIA, content rating).

- Web fallback: `apps/web/app/account-deletion/page.tsx` — static page describing the same flow with screenshots, contact mailbox for users who can't open the app.
